================================================================================
QAMP SHEET STRUCTURE REFERENCE
================================================================================
This document describes the sheet structure of each workbook in the QAMP system.
Column headers are documented as confirmed by logAllSheetHeaders() on 2026-06-05.

Canonical structure refers to the most recent fully-featured version of a sheet.
Older sheets may be missing columns added in later development cycles.

For workbook-to-script relationships, see ARCHITECTURE.md.
For function-level detail, see the *-Functions.md files.

  Cumulative Tracking
--------------------------------------------------------------------------------
    Student ID
    Last Billing Cycle
    Cumulative Hours Taught
    Cumulative Hours Billed
    Last Balance
    Last Updated

  One row per student. Stores the most recent cumulative data for every student
  who has ever been billed. When building a new billing cycle, consecutive
  returning students are populated via VLOOKUP from the previous cycle sheet.
  Students not found in the previous cycle (non-consecutive enrollment) are
  looked up here instead, allowing their historical data to survive enrollment
  gaps. New students with no record here are treated as first-time enrollees.

5. TEACHER INTEREST WORKBOOK
================================================================================
Attached script: Teacher Responses
Also read by: Contacts

--------------------------------------------------------------------------------
  New Teacher Responses (active form responses)
--------------------------------------------------------------------------------
  Current teacher interest form responses. Processed by handleTeacherFormSubmit().
  Headers are the raw Google Form question text — mapped to internal field names
  via TeacherFieldMap.

  Key fields include: Timestamp, First Name, Last Name, Email, Primary Instrument,
  Instruments willing to teach, availability, min/max students, school year
  interest, address, phone.

--------------------------------------------------------------------------------
  Returning Teacher Responses (active form responses)
--------------------------------------------------------------------------------
  Returning teacher form responses. Processed by handleReturningFormSubmit().
  Mapped via Returning Teacher FieldMap.

  Key fields include: Timestamp, First Name, Last Name, Email, Primary Instrument,
  availability, teaching preferences.

--------------------------------------------------------------------------------
  TeacherFieldMap
--------------------------------------------------------------------------------
    Form Header (from Form Responses)
    Internal Field Name (used in script)

  Maps new teacher form headers to internal field names.

--------------------------------------------------------------------------------
  Returning Teacher FieldMap
--------------------------------------------------------------------------------
    Form Header (from Form Responses)
    Internal Field Name (used in script)

  Maps returning teacher form headers to internal field names.

7. TEACHER ROSTER WORKBOOKS (per-teacher, per-year)
================================================================================
Attached script: Responses (creates/manages), Teacher Invoice (reads attendance),
                 Billing (reads balances)

  One workbook per teacher per academic year. Stored in Drive under the year's
  roster folder. File naming: "QAMP [Year] [LastName] [TeacherID]"
  URL stored in Teacher Roster Lookup sheet in Form Responses workbook.

--------------------------------------------------------------------------------
  [Season] Roster sheets (e.g. "Summer Roster", "Spring Roster", etc.)
--------------------------------------------------------------------------------
  One roster sheet per semester. Created by setupNewRosterTemplate() and
  populated by addStudentToRosterFromData().

  Columns (set by setupNewRosterTemplate()):
    Contacted
    First Lesson Date
    First Lesson Time
    Comments
    Last Name
    First Name
    Instrument
    Length
    Experience
    Grade
    School
    School Teacher
    Parent Last Name
    Parent First Name
    Phone
    Email
    Additional Contacts
    Hours Remaining
    Lessons Remaining
    Status
    Student ID
    Admin Comments
    System Comments

  Status values: active, carryover, dropped (lowercase, enforced by dropdown)

--------------------------------------------------------------------------------
  [Month] attendance sheets (e.g. "June", "May", etc.)
--------------------------------------------------------------------------------
  One sheet per month. Created by createMonthlyAttendanceSheet() in Utility.
  Row 1: column headers. Row 2: sign-off row (Month Complete / initials).
  Data starts at Row 3. Student sections separated by header rows.

  Columns:
    Student ID
    Student Name
    Date
    Length
    Status
    Comments
    Admin Review Date
    Invoice Date
    Payment Date
    Invoice Number
    Admin Comments

  Note: Attendance sheets created before the R2 sign-off row was introduced
  (old-style) start data at Row 2 with no sign-off row. Detection logic in
  processTeacherAttendanceForBilling() handles both styles.

--------------------------------------------------------------------------------
  Invoice Log
--------------------------------------------------------------------------------
  Created by createInvoiceLogSheet() in Responses. Tracks generated invoices
  per teacher. One sheet per teacher roster workbook.

    Invoice Number
    Invoice Date
    Invoice Period
    Invoice URL
    Total Amount
